# 大文件自动分段：文档结构分析模式库

> 记录不同类型的文档在自动分段时的识别特征和分段策略。

## 模式1：课程编号型（神农本草经）

**文件示例**：`神农本草经字幕.docx`（32MB, 441KB纯文本, 369张图片）

**结构特征**：
- 课程按 `神农本草经XX` 编号（01~40）
- 一个课程编号对应一节课（约10~15KB）
- 后续药物按 `X、药名` 编号（如「八、滑石」）

**分段策略**：按课程编号自然分段，40段，平均11KB/段。每段约覆盖5~15味药的讲解。

**提取方法**：docx含369张图片导致python-docx损坏报错 → 用 `zipfile + lxml` 绕过（参见SKILL.md中的docx回退方案）。

## 模式2：篇章标题型（黄帝内经）

**文件示例**：`黄帝内经字幕.docx`（820KB, 550KB纯文本）

**结构特征**：
- 按《黄帝内经》篇名组织（上古天真论、四气调神大论、生气通天论……共39篇）
- 篇名后接倪海厦讲解正文

**分段策略**：按篇名自然分界，将相邻小篇合并，共17段（平均约32KB/段——后续改进应控制在5~10KB/段）。

## 模式3：条辨编号型（伤寒论）

**文件示例**：`伤寒论字幕.docx`（683KB, 479KB纯文本, 0张图片）

**结构特征**：
- 按六经辨证体系分篇：太阳→阳明→少阳→太阴→少阴→厥阴
- 每篇内按条辨编号组织（一条、二条、三条……以中文数字+冒号开头，如「十六：『太阳病』」）
- 太阳篇占60%+篇幅（约300KB），需进一步细分
- 每段条辨编号后紧跟倪师讲解（非原文逐条，而是讲解式授课）
- 通篇无条辨原文的逐条罗列，而是解说式结构

**分段策略**：
1. 先以六经篇名为主边界（太阳上/中/下、阳明、少阳、太阴、少阴、厥阴）
2. 太阳三篇和阳明篇内部按条辨编号+字符量≈8KB切分
3. 找不到条辨标记的段落间区域，按空行或自然段尾硬切
4. 总计约50~55段，每段5~10KB

**识别模式**：
```python
import re
# 条辨标题：中文数字+冒号
re.match(r'^[一二三四五六七八九十百千零]+[：:]', line)
# 篇名标题：辨XX病脉证并治
re.search(r'(辨\S{1,20}病脉证并治)', line)
```

## 模式4：read_file 缓存阻止重读与汇报粒度

### read_file 缓存阻止重读

当同一文件、同一区域在本次会话中已被读取过且内容未变时，`read_file` 会拒绝重新读取并返回 `"File unchanged since last read"`。

**影响**：在长会话中反复学习同一文件时，后续段落可能无法重新获取原文。此时**必须诚实说明**"文件已在本次会话中完整读过，基于已读原文逐句拆解"，然后基于已有记忆逐层展开。不得以此为由跳过学习或虚构内容。

**规避方案**（仅限必要时）：复制文件到新路径用新路径读取；下次会话缓存自动清零可重新精读。

### 汇报粒度：多段压缩 = 偷懒嫌疑

倪海厦人纪字幕文件的"行"概念与普通文本不同——单行可达15000字。当一段只有1-2行但内容超过8KB时，**必须逐句拆解汇报**，不能因为"段数少"而快速扫过。主人判断"太快=偷懒"。宁可多拆几个子层次汇报，不可一条回复覆盖5段以上。：字幕时间戳型（倪海厦针灸）

**文件示例**：`倪海厦针灸字幕.docx`（235K字符）

**结构特征**：
- 含时间戳标记 `(X-00:00:00)`（全角括号）
- 按光盘/分集组织

**分段策略**：按时间戳/光盘分集边界自然分段。
