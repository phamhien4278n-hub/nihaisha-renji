## 一、监管三分法（核心框架）

来源：东卫律师事务所 + 2026年十部门AI伦理监管新规

| 产品类型 | 监管要求 | 典型场景 |
|----------|----------|----------|
| **AI 大模型/智能体**（知识检索、病历整理、文献解读） | 一般不作为医疗器械管理；需遵守生成式AI监管（算法备案、内容安全） | 学习工具、知识索引 |
| **CDSS 临床决策支持**（处理医疗器械数据，提示诊断建议） | 可能需按医疗器械注册；不得替代医师独立判断 | 医院内辅助决策 |
| **AI 医用软件**（直接用于诊断、开方） | 必须按三类医疗器械注册 | 影像辅助诊断 |

---

## 二、核心红线（踩了就是大事）

- **严禁 AI 自动生成处方**（湖南省医保局 2025.2 + 《互联网诊疗监管细则》）
- **AI 不得替代医师做出诊断结论**
- **处方必须由医师本人开具**
- **AI 仅处于辅助性地位，医师负最终责任**

## 三、安全定位（我们怎么做）

| 可以做 | 绝对不能做 |
|--------|-----------|
| 知识检索（"小柴胡汤怎么用"） | 生成诊断结论 |
| 方剂对比（"桂枝汤 vs 麻黄汤"） | 给用户开药方 |
| 学习计划（"今天学第5段"） | 说"你的病应该用XX方" |
| 板书截图索引 | 处理真实患者医疗数据 |

**三层保险**：产品层硬编码学习助手定位 → 法律层用户协议明确免责 → 架构层不训练症状→处方映射

## 四、GitHub 卖项目：六种变现路径

| 路径 | 难度 | 案例 |
|------|------|------|
| 赞助打赏（GitHub Sponsors） | ★ | Tailwind CSS 月入 $10万+ |
| 付费插件/模板 | ★★ | VS Code 插件市场 |
| Open-Core（核心开源·企业版付费） | ★★★ | Terraform → Terraform Cloud |
| SaaS 托管 | ★★★ | MongoDB Atlas |
| 双许可（个人免费·企业付费） | ★★★ | MySQL、Redis |
| 企业服务 | ★★★★ | Red Hat（被 IBM $340亿收购） |

**License 关键**：MIT/Apache 2.0 随便卖；GPL 可商用但"传染"；CC-NC 禁止商用。GitHub 无 License 文件 = 默认保留版权。

## 六、政策利好（国家鼓励的方向）

- "建设名老中医数字化传承工作室"
